# MIT License - What you need

If you want to publish this project under MIT on GitHub, keep these items:

1. LICENSE file in repo root (with MIT text).
2. Copyright line with your name and year.
3. Keep LICENSE in redistributed copies (required by MIT).
4. Mention license in README (recommended).
5. Separate asset licenses if images/audio are not fully yours.

## Quick checklist before public release

- [ ] LICENSE exists and has your real name.
- [ ] AUTHOR.md has your contact details.
- [ ] README has a "License" section.
- [ ] Third-party assets are listed with source + license.
- [ ] No secrets/API keys/private data in repository.
